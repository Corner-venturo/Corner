# I18N 規範

> ⚠️ **所有使用者看得到的中文，必須走 i18n，不要硬寫在 .tsx 裡**

## 快速範例

```tsx
// ❌ 錯誤
<Button>儲存</Button>
<span>載入中...</span>

// ✅ 正確
import { useTranslations } from 'next-intl'

function MyComponent() {
  const t = useTranslations('common')
  return (
    <>
      <Button>{t('save')}</Button>
      <span>{t('loading')}</span>
    </>
  )
}
```

## 新增翻譯 Key

1. 打開 `messages/zh-TW.json`
2. 找到適合的分類（或新增分類）
3. 加入 key-value
4. 在元件裡用 `t('分類.key')` 或 `useTranslations('分類')` + `t('key')`

## 分類結構

```
messages/zh-TW.json
├── common          # 通用按鈕、狀態文字
├── validation      # 表單驗證訊息
├── customer        # 客戶相關
├── order           # 訂單相關
├── tour            # 行程相關
├── passport        # 護照/證件相關
├── imageEditor     # 圖片編輯器
└── ...             # 依功能擴展
```

## 什麼要 i18n？

| 類型 | 要 i18n？ | 範例 |
|------|----------|------|
| 按鈕文字 | ✅ 是 | 儲存、取消、確認 |
| 標題/標籤 | ✅ 是 | 客戶資料、訂單列表 |
| 提示訊息 | ✅ 是 | 載入中...、儲存成功 |
| 錯誤訊息 | ✅ 是 | 此欄位必填 |
| placeholder | ✅ 是 | 請輸入姓名 |
| console.log | ❌ 否 | debug 用 |
| 註解 | ❌ 否 | // 這是註解 |
| 後端固定值 | ❌ 否 | database enum |

## 命名慣例

- 用 camelCase：`saveChanges` 不是 `save_changes`
- 語意化：`confirmDelete` 不是 `button1`
- 通用的放 `common`，專屬的放對應分類

## 檢查工具

```bash
# 找出 .tsx 裡的中文硬編碼（排除註解和 console）
grep -rn "[\u4e00-\u9fff]" src --include="*.tsx" | grep -v "//" | grep -v "console"
```
